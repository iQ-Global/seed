Overview

Seed is a minimal PHP framework designed for simplicity and developer freedom. It provides just enough structure to build robust applications without imposing unnecessary complexity.

Core Philosophy

  • Minimal but capable: Only what you need, nothing you don't
  • Easy to learn: Simple concepts, clear patterns
  • Developer freedom: Opinionated where it helps, flexible where it matters
  • Modern yet accessible: Works with PHP 7.0+ and modern development practices

üöÄ Quick Start

Installation

git clone https://github.com/iQ-Global/seed.git myproject
cd myproject
composer install
cp .env.example .env
./seed serve

Visit http://localhost:8000

Your First Route

1. Define route in app/routes.php:

<?php
$router->get('/hello/{name}', 'helloController/greet');

2. Create controller in app/controllers/helloController.php:

<?php
namespace App\Controllers;
use Seed\Core\Controller;

class helloController extends Controller {
    public function greet($name) {
        view('hello', ['name' => $name]);
    }
}

3. Create view in app/views/hello.php:

<h1>Hello, <?= esc($name) ?>!</h1>

That's it! Simple, clean, powerful.

üéØ Routing

Basic Routes

Define routes in app/routes.php using controller/function syntax:

<?php
// Basic routes
$router->get('/', 'homeController/index');
$router->post('/users', 'userController/store');
$router->put('/users/{id}', 'userController/update');
$router->delete('/users/{id}', 'userController/destroy');
$router->patch('/users/{id}', 'userController/patch');

Route Parameters

Use curly braces for dynamic segments:

<?php
$router->get('/user/{id}', 'userController/show');
$router->get('/posts/{postId}/comments/{commentId}', 'postController/showComment');

Parameters are passed to your controller method:

<?php
public function show($id) {
    // $id contains the value from the URL
}

Middleware

Apply middleware to routes for authentication, CSRF protection, etc.:

<?php
// Single middleware
$router->post('/profile', 'userController/update')
    ->middleware('auth');

// Multiple middleware
$router->post('/admin/users', 'adminController/createUser')
    ->middleware(['auth', 'csrf']);

Built-in Middleware: csrf, auth, rate_limit

Route Groups

Group routes with common middleware or prefixes:

<?php
// Group with middleware
$router->group(['middleware' => 'auth'], function($r) {
    $r->get('/dashboard', 'dashboardController/index');
    $r->get('/profile', 'userController/profile');
});

// Group with prefix
$router->group(['prefix' => 'api'], function($r) {
    $r->get('/users', 'apiController/users');
    $r->get('/posts', 'apiController/posts');
});

// Combine prefix and middleware
$router->group(['prefix' => 'admin', 'middleware' => 'auth'], function($r) {
    $r->get('/dashboard', 'adminController/dashboard');
    $r->get('/users', 'adminController/users');
});

üéÆ Controllers & Models

Controllers

Controllers handle HTTP requests. Create in app/controllers/:

<?php
namespace App\Controllers;
use Seed\Core\Controller;

class userController extends Controller {
    // Show all users
    public function index() {
        $users = db()->query("SELECT * FROM users");
        view('users/index', ['users' => $users]);
    }
    
    // Show single user
    public function show($id) {
        $user = db()->queryOne("SELECT * FROM users WHERE id = ?", [$id]);
        
        if (!$user) {
            return $this->response->notFound();
        }
        
        view('users/show', ['user' => $user]);
    }
    
    // Create user
    public function store() {
        // Validate input
        $rules = [
            'name' => 'required|min:3',
            'email' => 'required|email'
        ];
        
        if (!validate($this->request->post(), $rules)) {
            flash('error', 'Validation failed');
            return redirect_back();
        }
        
        // Save to database
        $id = db()->insert('users', $this->request->post());
        
        flash('success', 'User created!');
        redirect('/users/' . $id);
    }
}

Models

Models handle data and database logic. Create in app/models/:

<?php
namespace App\Models;
use Seed\Core\Model;

class userModel extends Model {
    protected $table = 'users';
    
    // Get all active users
    public function getActive() {
        return $this->query("SELECT * FROM {$this->table} WHERE status = ?", ['active']);
    }
    
    // Get user by ID
    public function find($id) {
        $result = $this->query("SELECT * FROM {$this->table} WHERE id = ?", [$id]);
        return $result[0] ?? null;
    }
    
    // Create user
    public function create($data) {
        return $this->insert($data);
    }
}

Use models in controllers:

$model = new \App\Models\userModel();
$users = $model->getActive();

üîê Authentication

Session-based authentication with password hashing.

Login Example

use Seed\Modules\Auth\Auth;

public function login() {
    $email = $this->request->post('email');
    $password = $this->request->post('password');
    
    // Find user
    $user = db()->queryOne("SELECT * FROM users WHERE email = ?", [$email]);
    
    // Verify password
    if ($user && Auth::verifyPassword($password, $user->password)) {
        Auth::login($user->id);
        flash('success', 'Logged in successfully!');
        redirect('/dashboard');
    }
    
    flash('error', 'Invalid credentials');
    redirect_back();
}

Logout

public function logout() {
    Auth::logout();
    flash('success', 'Logged out');
    redirect('/');
}

Check Authentication

// Using helper
if (is_logged_in()) {
    $userId = user_id();
}

// Using Auth class
if (Auth::check()) {
    $userId = Auth::id();
}

Protect Routes

$router->get('/dashboard', 'dashboardController/index')
    ->middleware('auth');

üíæ Storage & Files

Seed provides a clean file storage abstraction with multi-disk support.

Basic File Operations

// Store file
storage()->put('documents/file.txt', $contents);

// Get file contents
$contents = storage()->get('documents/file.txt');

// Check if exists
if (storage()->exists('documents/file.txt')) {
    // File exists
}

// Delete file
storage()->delete('documents/file.txt');

// Append to file
storage()->append('logs/app.log', "New log entry\n");

Multi-Disk Support

// Private storage (default)
storage()->put('private-file.txt', $data);

// Public storage (web-accessible)
storage('public')->put('image.jpg', $imageData);
$url = storage('public')->url('image.jpg');

// Uploads storage
storage('uploads')->put($filename, $fileData);

// Cache storage
storage('cache')->put('cache-key.json', $jsonData);

// Logs storage
storage('logs')->append('app.log', $logEntry);

Directory Operations

// Create directory
storage()->makeDirectory('documents/invoices');

// Delete directory
storage()->deleteDirectory('temp');

// List files
$files = storage()->files('documents');
$allFiles = storage()->files('documents', true); // Recursive

Quick Helpers: store_public() to store and get URL in one call, upload_path() for uploaded files

üîî Events

Simple event system for decoupled code and extensibility.

Register Listeners

// Register event listener
listen('user.created', function($data) {
    // Send welcome email
    email()
        ->to($data['email'])
        ->subject('Welcome!')
        ->body('Thanks for signing up')
        ->send();
});

Dispatch Events

// Trigger the event
event('user.created', [
    'email' => $user->email,
    'name' => $user->name
]);

Multiple Listeners

// Multiple listeners for same event
listen('order.placed', function($order) {
    // Send confirmation email
});

listen('order.placed', function($order) {
    // Update inventory
});

listen('order.placed', function($order) {
    // Log the order
});

event('order.placed', $orderData); // All 3 execute

Use Cases: Notifications, logging, cache invalidation, webhook triggers, updating related models

Framework Event Hooks

Seed dispatches these events automatically during its lifecycle:

seed.booting - Before framework init
seed.booted - After framework ready
request.received - When request arrives
view.rendering - Before view renders (can modify data!)
view.rendered - After view rendered
query.executing - Before database query
query.executed - After query completes
// Example: Log all queries
listen('query.executing', function($data) {
    log_debug('SQL: ' . $data['sql']);
});

// Example: Add global view data
listen('view.rendering', function($data) {
    return [
        'app_name' => config('app.name'),
        'year' => date('Y')
    ];
});

‚≠ê Best Practices

Security

  • ‚úì Always escape output: <?= esc($value) ?>
  • ‚úì Use prepared statements for all queries
  • ‚úì Enable CSRF protection for forms
  • ‚úì Validate all user input
  • ‚úì Hash passwords with Auth::hashPassword()

Database

  • ‚úì Use models for complex logic
  • ‚úì Use transactions for related operations
  • ‚úì Index frequently queried columns
  • ‚úì Never use raw queries without raw()

Code Organization

  • ‚úì Keep controllers thin (logic in models)
  • ‚úì Use descriptive names (mixedCase for files)
  • ‚úì Group related routes
  • ‚úì Use middleware for common functionality
  • ‚úì Create custom helpers for repeated code
