# Seed Framework Documentation

This directory contains three types of documentation for different use cases.

---

## Documentation Formats

### `text/` - AI Context & Quick Reference

**Purpose**: Complete text-based guide designed for AI context building and command-line reference.

**File**: `seed-framework-complete-guide.txt`

**Best for**:
- Loading into AI assistants for context
- Command-line reference (`less`, `grep`, `vim`)
- Offline documentation
- Quick text searches

**Contains**: All 20 topics with code examples in plain text format.

---

### `html/` - Human-Readable Documentation

**Purpose**: Beautiful, searchable, interactive documentation.

**File**: `index.html`

**Best for**:
- Learning the framework
- Browsing features
- Interactive search
- First-time users

**Features**:
- Tailwind CSS styling
- Sidebar navigation
- Live search functionality
- Responsive design
- Code syntax highlighting

**Usage**: Open `index.html` in your browser.

---

### `context/` - Reference Documentation

**Purpose**: Quick reference guides and documentation summaries.

**Files**:
- `documentation.md` - Documentation overview
- `quick-reference.md` - Essential commands and patterns

**Best for**:
- Quick lookups during development
- Common patterns and examples
- Essential command reference

---

### `examples/` - Configuration Files

**Purpose**: Working configuration examples.

**Files**:
- `nginx.conf` - Complete Nginx server configuration
- `.htaccess` - Apache configuration with security headers

**Best for**:
- Server setup
- Deployment configuration
- Copy-paste into your server

---

## Which Documentation Should I Use?

| Use Case | Documentation |
|----------|---------------|
| Learning the framework | **html/index.html** |
| AI-assisted development | **text/seed-framework-complete-guide.txt** |
| Quick command lookup | **context/quick-reference.md** |
| Server configuration | **examples/nginx.conf** or **examples/.htaccess** |
| Complete reference | **text/seed-framework-complete-guide.txt** |
